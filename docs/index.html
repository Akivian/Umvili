项目简介与背景

从教学与研究视角出发，解释 UMVILI 试图解决的问题与设计初衷。

项目动机

强化学习尤其是多智能体强化学习的训练过程与内部状态往往“不可见”， 学生和初学者难以直观理解算法如何学习、何时收敛以及为何失败。UMVILI 旨在通过可视化与交互式沙盘，为教学与研究提供一个“可观察、可解释”的环境。

目标与使用场景

  • 课堂演示强化学习 / 多智能体协作与竞争过程
  • 供学生完成课程项目或实验报告时使用
  • 研究者快速对比 IQL / QMIX 与规则型策略的行为与性能
  • 在复杂多资源环境下分析策略的泛化与稳定性

核心功能与特点

围绕“可视化”“多智能体”“教学友好”三大方向设计的功能模块。

多智能体与算法对比

  • 支持 IQL、QMIX 以及多种规则型智能体（保守型、探索型、自适应）
  • 同一环境中对比不同算法的收敛速度与最终表现
  • 支持多智能体之间的协作与竞争场景建模

复杂多资源环境

  • Sugar / Spice / Hazard 组成的多资源 Sugarscape 环境
  • 动态生成与消失的高价值资源与危险区
  • 支持资源竞争、风险规避与策略迁移等教学示例

丰富的可视化视图

  • 训练视图：Loss、Q 值、TD Error、探索率等多线曲线
  • 行为视图：动作分布、奖励趋势、策略熵
  • Q 值热图：在地图上直观展示策略价值分布
  • Network / Debug 视图：网络状态与性能指标监控

交互式实验配置

  • 通过 Experiment 视图配置算法组合、环境参数与超参数
  • 支持一键重置与场景切换，便于课堂演示与对比实验
  • 配置基于 JSON/YAML，可与命令行选项结合使用

使用指南与教程

面向学生与教师的分步骤上手指南，帮助快速在本地运行 UMVILI。

快速开始

  1. 安装 Python 3.8+ 与必要依赖（PyTorch、Pygame 等）。
  2. 克隆仓库并安装依赖：参考 README 中的 Installation 部分。
  3. 在命令行运行 python main.py 启动可视化界面。

详细步骤与配置示例可参考主仓库的 README.md 与 docs/zh/README.md。

键盘与交互操作速查

操作 说明
Space 暂停 / 继续模拟
R 重置当前实验
F 切换全屏模式
Tab / 点击界面标签 在 Overview / Training / Behavior / Debug / Experiment 视图间切换

技术细节与架构

面向研究者和工程师，概览 UMVILI 的模块划分、数据流与扩展方式。

技术栈概览

  • 语言：Python 3.8+
  • 深度学习：PyTorch（DQN / Double DQN / QMIX 等）
  • 可视化与 UI：Pygame 多视图界面
  • 数据处理：NumPy、Pandas
  • 配置管理：JSON / YAML + 命令行参数

整体架构（高层）

项目整体可分为四层：环境与智能体（Environment / Agents）、模拟与调度（MARLSimulation）、强化学习算法（MARL Agents & Trainers）、可视化系统（VisualizationSystem）。详细结构可参考 docs/ARCHITECTURE.md 与 docs/zh/ 下的中文文档。
